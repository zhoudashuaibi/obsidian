# ESLint 规则配置与自定义规则

## 📌 核心概念
ESLint 用于静态检查代码问题，不仅能统一风格，更重要的是提前发现潜在 bug。它通过解析 AST 匹配规则，规则既可以使用社区现成能力，也可以团队自定义实现。

## 💻 代码示例
```js
module.exports = {
  rules: {
    'no-console': 'warn',
    'no-debugger': 'error',
  },
};
```

## 🛠 实战场景
大型团队往往会把工程规范沉淀为 lint 规则，从“靠口头提醒”升级为“靠工具自动兜底”。

## 🎯 面试怎么问
- ESLint 的作用只是统一代码风格吗？
- 自定义规则通常怎么实现？
- 为什么要基于 AST 做 lint？

## ⚠️ 易错点 / 高阶拓展
Prettier 更偏格式化，ESLint 更偏语义与规则校验。两者职责要分清，否则配置容易互相打架。
