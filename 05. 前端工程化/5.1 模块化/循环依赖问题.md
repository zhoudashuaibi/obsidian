# 循环依赖问题

## 📌 核心概念
循环依赖指模块 A 依赖模块 B，同时模块 B 又反向依赖 A。它会让模块初始化顺序变复杂，CommonJS 与 ESM 的表现也不同，严重时会拿到未完成初始化的值。

## 💻 代码示例
```js
// a.js
import { b } from './b.js';
export const a = 'a';
console.log(b);

// b.js
import { a } from './a.js';
export const b = 'b';
console.log(a);
```

## 🛠 实战场景
大型项目拆模块时很容易因为 utils、store、常量、组件互相引用而形成环，最终造成运行时报错或奇怪的 `undefined`。

## 🎯 面试怎么问
- 什么是循环依赖？
- CommonJS 和 ESM 遇到循环依赖时行为有什么差异？
- 如何规避循环依赖？

## ⚠️ 易错点 / 高阶拓展
处理循环依赖通常靠重构边界，而不是硬改导入顺序。常见手段是抽公共层、反转依赖、拆状态与实现。
