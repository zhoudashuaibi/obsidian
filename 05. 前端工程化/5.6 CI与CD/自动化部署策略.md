# 自动化部署策略

## 📌 核心概念
自动化部署的目标不是“自动上线”这么简单，而是让不同环境的发布过程可重复、可回滚、可追踪。常见策略包括按分支部署、灰度发布、蓝绿部署、静态资源版本化。

```js
# 典型流程
# push main -> build -> upload artifact -> deploy to production
```

## 🛠 实战场景
多环境前端项目需要明确 dev、test、staging、prod 的部署边界，否则很容易出现“本地能跑、线上混乱”的问题。

## 🎯 面试怎么问
- 自动化部署常见策略有哪些？
- 为什么前端也需要灰度和回滚？
- 静态资源版本化有什么意义？

## ⚠️ 易错点 / 高阶拓展
部署策略和业务风险等级强相关。活动页、后台系统、核心 C 端应用的发布策略不应该一刀切。
