# 环境变量管理

## 📌 核心概念
环境变量用于区分不同运行环境下的配置，如接口地址、开关、第三方 key。工程化的关键不只是“会写 `.env`”，更重要的是分环境、分权限、避免泄漏和误用。

## 💻 代码示例
```js
VITE_API_BASE=/api
VITE_APP_ENV=production
```

## 🛠 实战场景
本地、测试、预发、生产环境经常对应不同服务地址和功能开关，变量管理不清会直接导致错连环境或泄露敏感信息。

## 🎯 面试怎么问
- 为什么要分环境变量？
- 前端环境变量为什么不能存真正机密？
- 构建时注入和运行时读取有什么区别？

## ⚠️ 易错点 / 高阶拓展
所有打进前端产物的变量都可能被用户看到，真正敏感配置必须留在服务端。这个点是高频追问。
