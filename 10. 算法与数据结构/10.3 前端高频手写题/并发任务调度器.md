# 并发任务调度器

## 📌 核心概念
并发调度器用于限制同时执行的异步任务数量，避免瞬时发起过多请求压垮浏览器、服务端或第三方接口。

## 💻 代码示例
```js
class Scheduler {
  constructor(limit) {
this.limit = limit
this.running = 0
this.queue = []
  }
}
```

## 🛠 实战场景
批量上传图片、批量请求详情页、抓取数据、CI 任务队列都常见。

## 🎯 面试怎么问
- 如何保证最大并发数不超过 limit？
- 任务失败后是否继续？
- 结果顺序和执行顺序如何解耦？

## ⚠️ 易错点 / 高阶拓展
很多人只会 `Promise.all`，但实际业务常需要限流、重试、取消和超时控制。
