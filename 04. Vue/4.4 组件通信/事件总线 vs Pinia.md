# 事件总线 vs Pinia

## 📌 核心概念
事件总线基于发布订阅，适合极少量临时跨组件通知；Pinia 是显式状态管理，适合可追踪、可维护的共享状态。随着项目复杂度上升，事件总线往往会让依赖关系变得隐蔽而混乱。

## 💻 代码示例
```js
// 事件总线思路
bus.emit('login', user);
bus.on('login', handler);

// Pinia 思路
const userStore = useUserStore();
userStore.setUser(user);
```

## 🛠 实战场景
小项目里事件总线写起来快，但中大型项目更看重状态来源清晰、调试方便、可测试性，因此通常更推荐 Pinia。

## 🎯 面试怎么问
- 事件总线为什么在大型项目里不推荐？
- Pinia 相比事件总线的优势是什么？
- 哪类场景仍可能使用事件总线？

## ⚠️ 易错点 / 高阶拓展
别把事件总线完全妖魔化。一次性通知、解耦非关键事件仍可使用，但不适合承载长期核心状态。
