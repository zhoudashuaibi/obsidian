# 插槽：默认、具名、作用域插槽

## 📌 核心概念
插槽让父组件向子组件预留区域插入内容。默认插槽提供基础内容分发，具名插槽支持多个位置，作用域插槽则允许子组件把数据回传给父组件决定如何渲染。

## 💻 代码示例
```js
<List>
  <template #item="{ row }">
    <strong>{{ row.name }}</strong>
  </template>
</List>
```

## 🛠 实战场景
组件库开发中，插槽是实现高可定制性的核心手段，比如表格列渲染、下拉项定制、空状态展示。

## 🎯 面试怎么问
- 默认插槽和具名插槽有什么区别？
- 什么是作用域插槽？
- 为什么说插槽提升了组件可扩展性？

## ⚠️ 易错点 / 高阶拓展
插槽不是“子传父”的替代品，它更像渲染能力下放。过度依赖复杂插槽也可能让组件 API 变得难懂。
