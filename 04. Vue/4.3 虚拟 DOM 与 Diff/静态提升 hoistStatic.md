# 静态提升 hoistStatic

## 📌 核心概念
Vue 3 编译阶段会把不会变化的静态节点提升到渲染函数外部复用，这叫静态提升。这样每次重新渲染时，无需重复创建这些 vnode，对性能很有帮助。

## 💻 代码示例
```js
// 编译前模板
// <div><p>hello</p><span>{{ count }}</span></div>

// 编译后大致会把 <p>hello</p> 提升为常量
```

## 🛠 实战场景
复杂模板中往往只有少量动态区域。静态提升能减少运行时创建 vnode 的成本，是 Vue 3 编译优化的重要一环。

## 🎯 面试怎么问
- 什么是静态提升？
- 它发生在编译期还是运行时？
- 静态节点提升后有什么收益？

## ⚠️ 易错点 / 高阶拓展
不要把它理解成 DOM 永远不更新，而是“编译器帮你把确定不变的描述对象提出来复用”。
