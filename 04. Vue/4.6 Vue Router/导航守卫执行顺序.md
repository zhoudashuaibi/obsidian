# 导航守卫执行顺序

## 📌 核心概念
Vue Router 提供全局守卫、路由独享守卫和组件内守卫。导航时它们按固定顺序执行，用于权限校验、数据预取、离开确认等逻辑。理解顺序能帮助你排查跳转异常和重复请求。

## 💻 代码示例
```js
router.beforeEach((to, from) => {
  if (!isLogin() && to.meta.auth) {
    return '/login';
  }
});
```

## 🛠 实战场景
后台系统鉴权、未保存表单离开提醒、埋点统计、动态标题设置都常依赖导航守卫。

## 🎯 面试怎么问
- 全局守卫、独享守卫、组件守卫谁先执行？
- Vue Router 4 为什么更推荐 return 而不是 `next`？
- 守卫里发请求要注意什么？

## ⚠️ 易错点 / 高阶拓展
守卫过重会拖慢首屏和路由切换。权限逻辑应尽量清晰集中，避免多层守卫相互覆盖。
