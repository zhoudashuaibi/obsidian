# 路由懒加载实现

## 📌 核心概念
路由懒加载通常通过动态导入 `() => import(...)` 实现，构建工具会把页面组件拆成独立 chunk，只有访问对应路由时才下载，从而减少首屏资源体积。

## 💻 代码示例
```js
const routes = [
  {
    path: '/about',
    component: () => import('../views/AboutView.vue'),
  },
];
```

## 🛠 实战场景
页面多、模块大的后台系统和中后台项目特别适合用路由懒加载，能明显改善初始加载速度。

## 🎯 面试怎么问
- 路由懒加载底层依赖什么机制？
- 为什么它能优化首屏？
- 懒加载会不会影响切换体验？

## ⚠️ 易错点 / 高阶拓展
懒加载减少的是首屏包体，不是总资源体积。若切页卡顿明显，还要结合预加载、骨架屏和缓存策略。
