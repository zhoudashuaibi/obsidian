# 动态路由与权限控制

## 📌 核心概念
动态路由是指根据用户角色、后端返回菜单或运行时条件动态注册路由。常用于后台权限系统，让不同用户在登录后拿到不同可访问页面集合。

## 💻 代码示例
```js
const asyncRoutes = getRoutesByRole(role);
asyncRoutes.forEach((route) => {
  router.addRoute(route);
});
```

## 🛠 实战场景
管理系统经常需要根据角色、租户、菜单配置动态控制页面访问范围，这时静态路由表就不够用了。

## 🎯 面试怎么问
- 为什么权限系统常用动态路由？
- 前端权限控制和后端权限校验是什么关系？
- 动态添加路由后刷新页面要注意什么？

## ⚠️ 易错点 / 高阶拓展
前端权限控制更多是体验层和导航层限制，真正安全仍要靠后端接口校验。不要把前端隐藏菜单当成完整权限方案。
