# React Native vs Flutter 对比

## 📌 核心概念
React Native 通过 JS 驱动原生组件或新架构桥接，Flutter 使用 Dart 和自绘渲染引擎。前者更贴近 Web/React 生态，后者在 UI 一致性和渲染控制上更强。

## 💻 代码示例
```text
React Native: JS/TS + React + Native Bridge
Flutter: Dart + Skia + Widget Tree
```

## 🛠 实战场景
技术选型时常从团队技能、性能要求、平台一致性、原生协作成本几个维度做比较。

## 🎯 面试怎么问
- RN 和 Flutter 的渲染方式差异？
- 为什么 Flutter UI 一致性更强？
- Web 团队切入哪种成本更低？

## ⚠️ 易错点 / 高阶拓展
不要把跨端框架选择只看性能。组织能力、招聘成本、已有技术栈往往更决定长期收益。
