# setData 性能优化

## 📌 核心概念
setData 会把逻辑层数据变更同步到渲染层，频繁调用或传输大对象会带来通信和渲染开销。优化核心是减少次数、缩小数据量、按需更新。

## 💻 代码示例
```js
this.setData({
  ['list[3].done']: true
})
```

## 🛠 实战场景
长列表勾选、输入联想、拖拽排序等高频交互场景最容易暴露 setData 性能问题。

## 🎯 面试怎么问
- 为什么 setData 不能传太大对象？
- 局部更新为什么更高效？
- 如何减少频繁 setData？

## ⚠️ 易错点 / 高阶拓展
把整个页面状态对象一次次整包 setData，是小程序性能问题的高发源。
