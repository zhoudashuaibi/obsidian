# 小程序双线程架构

## 📌 核心概念
微信小程序采用逻辑层和渲染层双线程架构。逻辑层运行 JS，渲染层负责 WXML/WXSS，二者通过桥接通信，避免直接操作 DOM。

## 💻 代码示例
```text
AppService(JSCore) <-> Native Bridge <-> WebView(Render)
```

## 🛠 实战场景
解释为什么小程序不能像网页一样直接操作 DOM、为什么 setData 过大有性能问题，通常都会提到双线程架构。

## 🎯 面试怎么问
- 为什么小程序要做双线程隔离？
- 逻辑层和视图层怎么通信？
- 双线程架构带来的限制是什么？

## ⚠️ 易错点 / 高阶拓展
小程序不是“阉割版网页”，它的运行时模型从一开始就和浏览器不同。
