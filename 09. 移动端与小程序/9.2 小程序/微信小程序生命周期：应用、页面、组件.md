# 微信小程序生命周期：应用、页面、组件

## 📌 核心概念
小程序有应用级、页面级、组件级三层生命周期。理解它们的触发时机，有助于正确放置初始化、数据拉取、事件订阅和销毁逻辑。

## 💻 代码示例
```js
App({ onLaunch() {}, onShow() {} })
Page({ onLoad() {}, onShow() {}, onUnload() {} })
Component({ lifetimes: { attached() {}, detached() {} } })
```

## 🛠 实战场景
页面切换、返回前台、组件挂载时重复请求或状态错乱，往往是生命周期使用不当导致。

## 🎯 面试怎么问
- onLoad 和 onShow 的区别？
- 组件 attached 和 page onLoad 谁先执行？
- 哪些逻辑适合放在 App 级？

## ⚠️ 易错点 / 高阶拓展
不要把所有初始化都塞到 onShow。页面返回时 onShow 会重复触发，容易造成重复请求。
