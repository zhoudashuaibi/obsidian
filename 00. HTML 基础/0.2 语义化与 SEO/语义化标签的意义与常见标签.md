# 语义化标签的意义与常见标签

## 📌 核心概念
语义化标签通过标签本身表达内容角色和结构，如 `header`、`main`、`article`、`nav`、`section`、`footer`。它提升可读性、可访问性，也有助于搜索引擎理解页面结构。

## 💻 代码示例
```html
<header>站点头部</header>
<nav>主导航</nav>
<main>
  <article>文章内容</article>
  <aside>侧边栏</aside>
</main>
<footer>页脚</footer>
```

## 🛠 实战场景
企业官网、博客、内容页和组件库文档都适合坚持语义化结构，这不仅是规范问题，也影响 SEO 与无障碍体验。

## 🎯 面试怎么问
- 什么是 HTML 语义化？
- 为什么不用一堆 `div`？
- 常见语义化标签分别适合什么场景？

## ⚠️ 易错点 / 高阶拓展
语义化不等于乱换标签名，关键是“结构角色是否匹配”。错误使用 `section`、`article` 反而会降低可读性。
