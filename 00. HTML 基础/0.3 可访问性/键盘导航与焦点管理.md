# 键盘导航与焦点管理

## 📌 核心概念
可访问页面不能只靠鼠标操作。键盘导航要求用户能通过 Tab、Enter、Esc、方向键等完成核心交互，而焦点管理则确保当前可操作位置清晰可感知。

## 💻 代码示例
```html
<button>打开弹窗</button>
<div role="dialog" aria-modal="true" tabindex="-1">
  <button>确认</button>
  <button>取消</button>
</div>
```

## 🛠 实战场景
弹窗打开后焦点应移入弹窗，关闭后回到触发按钮；菜单、轮播、Tabs 也都需要合理的键盘交互设计。

## 🎯 面试怎么问
- 什么是焦点陷阱？
- 为什么自定义组件要考虑键盘导航？
- `tabindex` 的常见取值分别意味着什么？

## ⚠️ 易错点 / 高阶拓展
很多前端只关注视觉效果，忽略键盘用户。真正的可访问性要求交互路径和焦点流转都能闭环。
