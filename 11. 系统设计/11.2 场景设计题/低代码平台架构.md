# 低代码平台架构

## 📌 核心概念
低代码平台通常分物料体系、可视化编辑器、状态管理、渲染引擎、Schema 协议、出码或运行时承载等模块。核心是把页面描述抽象成可编辑、可存储、可渲染的结构化数据。

## 💻 代码示例
```json
{
  "component": "Button",
  "props": { "type": "primary", "text": "提交" },
  "children": []
}
```

## 🛠 实战场景
表单搭建、营销页搭建、运营后台配置化页面是低代码最常见落地场景。

## 🎯 面试怎么问
- Schema 驱动为什么是低代码核心？
- 编辑器态和运行时如何解耦？
- 低代码平台的边界在哪？

## ⚠️ 易错点 / 高阶拓展
低代码并不意味着可以覆盖所有复杂业务。越靠近高度定制和复杂交互，平台抽象成本越高。
