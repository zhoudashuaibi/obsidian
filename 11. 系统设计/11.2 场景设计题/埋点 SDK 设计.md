# 埋点 SDK 设计

## 📌 核心概念
埋点 SDK 设计要解决事件采集、公共上下文注入、离线缓存、批量上报、失败重试、性能影响和可扩展性。

## 💻 代码示例
```js
tracker.track('button_click', {
  page: location.pathname,
  buttonId: 'submit'
})
```

## 🛠 实战场景
增长分析、用户行为分析、A/B 实验、错误追踪都依赖稳定的埋点基础设施。

## 🎯 面试怎么问
- SDK 为什么要批量上报？
- 页面卸载时如何提高送达率？
- 如何设计事件模型和版本兼容？

## ⚠️ 易错点 / 高阶拓展
埋点过多会反噬页面性能和数据质量，采集前要明确事件模型和字段治理。
