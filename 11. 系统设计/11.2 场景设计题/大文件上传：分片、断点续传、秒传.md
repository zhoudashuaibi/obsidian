# 大文件上传：分片、断点续传、秒传

## 📌 核心概念
大文件上传通常会把文件切片并并发上传，服务端按 hash + chunk index 合并。断点续传依赖已上传分片记录，秒传依赖文件指纹命中服务端已有资源。

## 💻 代码示例
```js
const chunkSize = 2 * 1024 * 1024
for (let start = 0; start < file.size; start += chunkSize) {
  chunks.push(file.slice(start, start + chunkSize))
}
```

## 🛠 实战场景
视频、安装包、设计稿等大文件上传是系统设计高频题。

## 🎯 面试怎么问
- 为什么要分片？
- 断点续传需要哪些服务端能力？
- 秒传是否真的不上传任何数据？

## ⚠️ 易错点 / 高阶拓展
秒传不是只比文件名，通常至少要有 hash、大小等校验；否则会误判同名文件。
