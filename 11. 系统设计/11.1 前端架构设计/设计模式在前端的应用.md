# 设计模式在前端的应用

## 📌 核心概念
前端常见设计模式包括单例、发布订阅、观察者、策略、工厂、装饰器等。它们的价值不在背定义，而在于提升复用、解耦和扩展能力。

## 💻 代码示例
```js
const strategies = {
  phone: (v) => /^1\d{10}$/.test(v),
  email: (v) => /@/.test(v),
}
function validate(type, value) {
  return strategies[type](value)
}
```

## 🛠 实战场景
表单校验、弹窗管理、事件总线、埋点插件体系都能落到设计模式。

## 🎯 面试怎么问
- 前端里哪些场景适合策略模式？
- 发布订阅和观察者有什么区别？
- 单例模式在前端的风险是什么？

## ⚠️ 易错点 / 高阶拓展
模式只是经验总结，不要为了套模式而套模式。能用简单函数解决的问题，没必要强行上复杂抽象。
