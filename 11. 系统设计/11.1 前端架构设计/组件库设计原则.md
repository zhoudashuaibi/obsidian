# 组件库设计原则

## 📌 核心概念
组件库设计要平衡复用性、一致性、可扩展性和可维护性。好的组件库不仅有 UI 组件，还要有 API 设计规范、主题能力、文档和测试体系。

## 💻 代码示例
```jsx
<Button type="primary" size="large" loading>提交</Button>
```

## 🛠 实战场景
中后台、B 端平台、多业务线统一视觉与交互规范时，组件库是基础设施。

## 🎯 面试怎么问
- 设计一个 Button 组件会考虑哪些点？
- 组件库如何做主题定制？
- 业务组件和基础组件如何分层？

## ⚠️ 易错点 / 高阶拓展
不要一开始就堆大量“万能参数”。高可配置不等于好设计，过度抽象反而让组件难用难维护。
