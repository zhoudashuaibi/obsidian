# 存储容量与跨域限制

## 📌 核心概念
浏览器存储按 origin 隔离，不同协议、域名、端口通常互不共享。不同存储方案容量上限也不同，且会受到隐私模式、浏览器策略和配额回收影响。

## 💻 代码示例
```text
https://a.example.com !== https://b.example.com
https://example.com !== http://example.com
https://example.com:443 !== https://example.com:8443
```

## 🛠 实战场景
多子域系统、主站与嵌入页通信、离线数据缓存、浏览器配额报警都需要理解 origin 隔离。

## 🎯 面试怎么问
- 同一站点下哪些条件变化会导致存储隔离？
- IndexedDB 配额不够时会怎样？
- iframe 能否直接访问别域 localStorage？

## ⚠️ 易错点 / 高阶拓展
不要把“同一个一级域名”理解成可共享所有存储。Cookie 可通过 domain 做一定范围共享，但 localStorage/sessionStorage/IndexedDB 按 origin 隔离。
