# HTTPS 中间人攻击

## 📌 核心概念
中间人攻击是攻击者位于通信两端之间，窃听、篡改或伪造数据。HTTPS 通过证书链、CA 信任体系和密钥协商降低这种风险。

## 💻 代码示例
```text
用户 <-> 攻击者 <-> 服务器
没有证书校验: 可被伪装
有证书校验: 伪造证书会触发浏览器告警
```

## 🛠 实战场景
公共 Wi-Fi、恶意代理、公司网关注入、抓包调试工具都能引出中间人攻击这个话题。

## 🎯 面试怎么问
- HTTPS 如何防止中间人攻击？
- 为什么安装根证书后抓包工具能解密 HTTPS？
- 证书钉扎适合什么场景？

## ⚠️ 易错点 / 高阶拓展
HTTPS 不是绝对防监听。如果客户端主动信任了攻击者证书，链路依然可能被解密。
