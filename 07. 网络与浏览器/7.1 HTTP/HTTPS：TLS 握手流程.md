# HTTPS：TLS 握手流程

## 📌 核心概念
HTTPS = HTTP + TLS。TLS 握手的目标是协商加密套件、完成身份校验、生成会话密钥。现代浏览器通常用 TLS 1.3，握手轮次更少，前向安全默认开启。

## 💻 代码示例
```text
ClientHello -> 支持的版本/加密套件/随机数
ServerHello -> 选定算法/证书/随机数
验证证书 -> 生成共享密钥
Finished -> 后续 HTTP 数据开始加密传输
```

## 🛠 实战场景
登录、支付、后台管理等场景都依赖 TLS。面试常结合“浏览器如何确认服务端身份”“为什么抓包看到的是密文”一起追问。

## 🎯 面试怎么问
- TLS 握手核心做了哪几件事？
- 证书校验失败会发生什么？
- TLS 1.2 和 TLS 1.3 的主要差异是什么？

## ⚠️ 易错点 / 高阶拓展
不要把“非对称加密传输全部数据”当成 HTTPS 原理。非对称主要用于身份校验和密钥协商，业务数据通常还是走对称加密。
