# 作用域链与词法作用域

## 📌 核心概念
词法作用域指函数在定义时就确定了可访问的外层变量范围，而不是调用时决定；多个词法环境串联起来就是作用域链。查找变量时会从当前作用域逐层向上查，直到全局。

## 💻 代码示例
```js
const color = 'global';

function outer() {
  const color = 'outer';

  function inner() {
    console.log(color); // outer
  }

  return inner;
}

const fn = outer();
fn();
```

## 🛠 实战场景
模块封装、组件私有变量、Hook/闭包缓存本质上都建立在词法作用域之上。理解它后，才能解释“为什么函数拿到的是定义处环境”。

## 🎯 面试怎么问
- 什么是词法作用域？
- 作用域链的查找过程是怎样的？
- 为什么函数换个地方调用，拿到的外部变量不变？

## ⚠️ 易错点 / 高阶拓展
很多人会把作用域和执行上下文混淆：作用域决定“能访问谁”，执行上下文决定“什么时候创建、里面有什么”。`with` 和 `eval` 会让静态分析变差。
