# 闭包的定义与内存泄漏风险

## 📌 核心概念
闭包是函数与其定义时词法环境的组合。只要内部函数仍被引用，外层作用域中的变量就可能继续存活。它既是封装能力来源，也是内存泄漏高发点。

## 💻 代码示例
```js
function createCounter() {
  let count = 0;

  return function () {
    count += 1;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

## 🛠 实战场景
防抖、节流、模块私有状态、缓存函数都依赖闭包。但如果把大量 DOM、定时器、旧数据放进闭包且不释放，就会让对象长期无法被回收。

## 🎯 面试怎么问
- 闭包到底是什么？
- 闭包为什么能访问外层变量？
- 闭包一定会导致内存泄漏吗？

## ⚠️ 易错点 / 高阶拓展
闭包本身不是问题，问题在于“无用引用没有断开”。典型场景包括事件监听未卸载、定时器未清理、缓存无限增长。
