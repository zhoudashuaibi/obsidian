# 常见内存泄漏场景

## 📌 核心概念
内存泄漏的本质是“本该无用的对象仍然可达”。前端高频场景包括未清理定时器、事件监听器、闭包缓存、全局变量、DOM 脱离文档后仍被引用、Map 缓存无限增长。

## 💻 代码示例
```js
function mount() {
  const handler = () => console.log('resize');
  window.addEventListener('resize', handler);

  return () => {
    window.removeEventListener('resize', handler);
  };
}
```

## 🛠 实战场景
SPA 页面切换、弹窗反复挂载销毁、图表组件和大列表最容易出现泄漏；问题通常表现为页面越用越卡、内存曲线只涨不降。

## 🎯 面试怎么问
- 前端最常见的内存泄漏有哪些？
- 如何定位某个组件是否泄漏？
- 为什么闭包和事件监听器容易一起出问题？

## ⚠️ 易错点 / 高阶拓展
排查要结合 Chrome Memory/Performance、快照对比、保留路径分析。不要把“内存上涨”直接等同于泄漏，要先区分缓存、正常波动和真正不可释放对象。
