# 垃圾回收算法：标记清除与引用计数

## 📌 核心概念
现代 JavaScript 引擎以标记清除为主：从根对象出发标记可达对象，未标记的即视为垃圾并回收。引用计数是早期思路，对象被引用次数归零时回收，但无法处理循环引用。

## 💻 代码示例
```js
function createData() {
  const cache = { list: new Array(1000).fill('x') };
  return () => cache.list.length;
}

let fn = createData();
console.log(fn());
fn = null; // 断开引用后，闭包内数据才有机会被回收
```

## 🛠 实战场景
性能优化、内存泄漏排查、理解闭包和 DOM 引用长期驻留，都需要知道对象为什么“还活着”以及什么时候能被回收。

## 🎯 面试怎么问
- 标记清除的核心流程是什么？
- 为什么引用计数会被循环引用卡住？
- 为什么把变量设为 `null` 只是“有机会”被回收？

## ⚠️ 易错点 / 高阶拓展
GC 时机由引擎决定，不是代码里 `delete` 或 `null` 就会立刻释放内存。高阶追问会延伸到分代回收、新生代/老生代。
