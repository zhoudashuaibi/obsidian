# 并发控制：限制并发数实现

## 📌 核心概念
并发控制指同时运行的异步任务数量受限，而不是一口气全部发出。它的核心是维护任务队列和“运行中数量”，在任务完成后继续补位。

## 💻 代码示例
```js
async function runPool(tasks, limit) {
  const results = [];
  const executing = new Set();

  for (const task of tasks) {
    const p = Promise.resolve().then(task).then((res) => {
      results.push(res);
      executing.delete(p);
    });

    executing.add(p);
    if (executing.size >= limit) {
      await Promise.race(executing);
    }
  }

  await Promise.all(executing);
  return results;
}
```

## 🛠 实战场景
批量上传、图片预加载、爬虫抓取、消息推送都需要限制并发，否则容易打满浏览器连接数、把后端压垮，或触发第三方接口限流。

## ⚠️ 易错点 / 高阶拓展
注意“限制并发数”不等于“节流”。真正上线时还要考虑失败重试、取消、超时、优先级和结果回填顺序。
