# 继承的六种实现方式

## 📌 核心概念
经典面试会考原型链继承、借用构造函数、组合继承、原型式继承、寄生组合继承，以及 ES6 class extends。核心关注点是：实例属性是否共享、方法是否复用、调用父构造函数次数是否合理。

## 💻 代码示例
```js
function Parent(name) {
  this.name = name;
  this.colors = ['red'];
}

Parent.prototype.say = function () {
  return this.name;
};

function Child(name) {
  Parent.call(this, name);
}

Child.prototype = Object.create(Parent.prototype);
Child.prototype.constructor = Child;
```

## 🛠 实战场景
老代码迁移、阅读早期框架源码时仍会碰到 ES5 继承写法；理解它们的优缺点有助于解释为什么现代项目更偏向 class 或组合优于继承。

## 🎯 面试怎么问
- 常见继承方案有哪些？
- 为什么组合继承会调用两次父构造函数？
- 为什么说寄生组合继承是 ES5 时代较优方案？

## ⚠️ 易错点 / 高阶拓展
不要只背名字，要能说出每种方案对“引用属性共享”和“原型方法复用”的影响。现代业务代码更推荐组合、委托、函数式封装。
