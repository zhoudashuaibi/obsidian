# 原型链机制

## 📌 核心概念
对象查找属性时，先查自身，再沿着 `[[Prototype]]` 向上查找，直到 `null` 为止，这条查找链就是原型链。JavaScript 大量“继承”能力都建立在这套委托机制上。

## 💻 代码示例
```js
const animal = {
  eat() {
    console.log('eat');
  },
};

const dog = Object.create(animal);
dog.bark = () => console.log('wang');

dog.eat(); // 来自原型
console.log(Object.getPrototypeOf(dog) === animal); // true
```

## 🛠 实战场景
手写类库、理解数组方法从哪来、排查属性遮蔽问题时，都必须知道属性查找不是复制，而是沿链委托。

## 🎯 面试怎么问
- 什么是原型链？
- 对象读取属性时的查找顺序是什么？
- `Object.create` 做了什么？

## ⚠️ 易错点 / 高阶拓展
原型链过长会增加查找成本，但真正性能瓶颈通常不在这里。更重要的是分清“实例自有属性”和“原型共享方法”。
