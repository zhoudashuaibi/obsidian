# 可选链、空值合并、逻辑赋值运算符

## 📌 核心概念
可选链 `?.` 用于安全访问可能为空的属性或方法；空值合并 `??` 只在左侧为 `null` 或 `undefined` 时取右值；逻辑赋值如 `??=`、`||=`、`&&=` 则把“判断再赋值”写得更短。

## 💻 代码示例
```js
const user = { profile: { name: 'Tom' } };
console.log(user.profile?.name); // Tom
console.log(user.address?.city); // undefined

const page = 0 ?? 1; // 0
const title = '' || 'default'; // default

const config = {};
config.timeout ??= 3000;
console.log(config.timeout); // 3000
```

## 🛠 实战场景
处理后端不稳定字段、表单默认值、配置兜底时，这组语法能显著减少冗长判空代码，也更不容易误伤 `0`、``、`false` 这类合法值。

## 🎯 面试怎么问
- `||` 和 `??` 的区别是什么？
- 可选链为什么不能出现在赋值左侧？
- `??=` 适合什么默认值场景？

## ⚠️ 易错点 / 高阶拓展
很多 bug 来自把 `0`、空字符串误当“无值”。默认值场景优先考虑 `??`，而不是习惯性使用 `||`。
