# 解构赋值高级用法

## 📌 核心概念
解构赋值不仅能做基本拆值，还支持默认值、重命名、嵌套解构、剩余参数、函数参数解构。它让取值更简洁，但复杂层级下也可能牺牲可读性。

## 💻 代码示例
```js
const user = {
  profile: { name: 'Tom', city: 'Hangzhou' },
  score: 99,
};

const {
  profile: { name, city },
  score: point = 0,
} = user;

function init({ page = 1, pageSize = 10 } = {}) {
  return [page, pageSize];
}
```

## 🛠 实战场景
接口响应解析、函数参数默认配置、数组交换、组件 props 读取都会频繁使用解构，让数据读取更贴近业务语义。

## 🎯 面试怎么问
- 解构默认值何时生效？
- 重命名和嵌套解构怎么写？
- 为什么给函数参数解构时常配合 `= {}`？

## ⚠️ 易错点 / 高阶拓展
默认值只在值严格等于 `undefined` 时触发；过深解构不利于维护，容易在接口字段变化时直接抛错。
