# 迭代器协议与可迭代协议

## 📌 核心概念
可迭代协议要求对象实现 `Symbol.iterator` 方法，并返回迭代器；迭代器协议要求返回对象具备 `next()`，每次产出 `{ value, done }`。`for...of`、展开运算符都基于这套协议。

## 💻 代码示例
```js
const range = {
  start: 1,
  end: 3,
  [Symbol.iterator]() {
    let current = this.start;
    const end = this.end;
    return {
      next() {
        if (current <= end) {
          return { value: current++, done: false };
        }
        return { value: undefined, done: true };
      },
    };
  },
};

console.log([...range]); // [1, 2, 3]
```

## 🛠 实战场景
如果你想让自定义数据结构支持 `for...of`、`Array.from`、解构和扩展语法，就需要实现可迭代协议。

## 🎯 面试怎么问
- 什么对象可以被 `for...of` 遍历？
- 迭代器协议和可迭代协议有什么区别？
- 数组为什么既是可迭代对象，又能返回迭代器？

## ⚠️ 易错点 / 高阶拓展
可迭代对象不一定自己就是迭代器；很多人会把 `Symbol.iterator` 返回值和对象本身混为一谈。
