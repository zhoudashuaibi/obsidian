# Node.js 内存泄漏排查方法

## 📌 核心概念
Node.js 内存泄漏通常表现为堆内存持续增长、GC 频繁但回收无效。排查要结合堆快照、对象引用链、定时器、缓存、闭包和事件监听器。

## 💻 代码示例
```bash
node --inspect app.js
node --trace-gc app.js
// DevTools 中对比 Heap Snapshot
```

## 🛠 实战场景
线上服务运行数小时后 OOM、接口越来越慢、实例频繁重启，基本都要走内存排查流程。

## ⚠️ 易错点 / 高阶拓展
内存上涨不一定都是泄漏，也可能是缓存策略合理增长。关键是看对象是否应该被释放却仍被引用。
