# 中间件洋葱模型

## 📌 核心概念
洋葱模型指请求从外到内依次进入中间件，响应再从内到外逐层返回。Koa 通过 async/await + next 天然体现这一结构。

## 💻 代码示例
```js
app.use(async (ctx, next) => {
  console.log(1)
  await next()
  console.log(4)
})
app.use(async (ctx, next) => {
  console.log(2)
  await next()
  console.log(3)
})
```

## 🛠 实战场景
鉴权、日志、异常处理、响应包装都很适合放在中间件链上。

## 🎯 面试怎么问
- 为什么 Koa 更容易体现洋葱模型？
- next 调用前后各适合做什么？
- Express 和 Koa 中间件模型差异在哪？

## ⚠️ 易错点 / 高阶拓展
忘记 `await next()` 会直接打断调用链，某些后置逻辑也可能不执行。
