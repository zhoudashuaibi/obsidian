# 错误处理中间件

## 📌 核心概念
错误处理中间件负责统一捕获同步异常、异步 reject、业务错误并做日志、告警、格式化响应。框架不同，接入方式也不同。

## 💻 代码示例
```js
app.use(async (ctx, next) => {
  try {
await next()
  } catch (err) {
ctx.status = err.status || 500
ctx.body = { message: err.message }
  }
})
```

## 🛠 实战场景
企业项目通常会统一拦截数据库错误、参数校验错误、第三方服务超时。

## 🎯 面试怎么问
- Express 的错误处理中间件签名是什么？
- 为什么异步错误需要额外处理？
- 统一错误响应层有什么价值？

## ⚠️ 易错点 / 高阶拓展
只捕获同步 throw 不够，Promise reject、流错误、未处理异常都需要单独纳管。
