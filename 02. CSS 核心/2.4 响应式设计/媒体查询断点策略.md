# 媒体查询断点策略

## 📌 核心概念
媒体查询让样式根据视口、设备特征切换。断点设计不应死背设备型号，而应围绕内容布局什么时候“开始挤压、换行、难看”来设定。常见策略是移动优先，从小屏逐步增强。

## 💻 代码示例
```css
.grid {
  display: grid;
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1200px) {
  .grid {
    grid-template-columns: repeat(4, 1fr);
  }
}

```

## 🛠 实战场景
官网、后台、活动页都需要在手机、平板、桌面之间平滑切换。断点应该服务于内容，而不是死套某台设备分辨率。

## 🎯 面试怎么问
- 媒体查询常见写法有哪些？
- 为什么推荐移动优先？
- 断点应该按设备划分还是按内容划分？

## ⚠️ 易错点 / 高阶拓展
不要堆太多零碎断点，否则维护成本很高。高阶会关注容器查询 `@container` 是否能替代部分媒体查询职责。
