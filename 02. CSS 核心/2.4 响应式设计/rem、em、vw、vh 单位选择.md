# rem、em、vw、vh 单位选择

## 📌 核心概念
`rem` 基于根元素字体大小，适合全局尺寸缩放；`em` 基于当前元素字体大小，适合局部相对缩放；`vw`、`vh` 基于视口尺寸，适合全屏布局和流体尺寸，但也容易在移动端踩坑。

## 💻 代码示例
```css
:root {
  font-size: 16px;
}

.title {
  font-size: 2rem;
}

.button {
  padding: 0.75em 1.5em;
}

.hero {
  min-height: 100vh;
  font-size: 4vw;
}
```

## 🛠 实战场景
设计稿适配、字体系统、全屏首屏、移动端比例布局都需要在这些单位中权衡，关键是知道“参考系是谁”。

## 🎯 面试怎么问
- `rem` 和 `em` 的区别是什么？
- `vw/vh` 在移动端有哪些坑？
- 什么时候更适合用 `px` 而不是相对单位？

## ⚠️ 易错点 / 高阶拓展
移动端地址栏伸缩会让 `100vh` 不稳定，现代可以关注 `svh`、`lvh`、`dvh`。面试中答出“参考系”就已经很加分。
