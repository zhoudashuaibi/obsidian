# 合成层 Compositing Layer

## 📌 核心概念
浏览器渲染并非所有元素都在同一层完成。满足某些条件的元素会被提升到独立合成层，之后可由合成线程单独处理位置、透明度等变化，从而减少主线程参与。

## 💻 代码示例
```css
.panel {
  transform: translateZ(0);
}

.modal {
  position: fixed;
  opacity: 0.98;
}
```

## 🛠 实战场景
弹窗、视频、Canvas、复杂动画元素有时会因为独立层而获得更流畅的渲染表现，但层数过多也会带来显存和管理成本。

## 🎯 面试怎么问
- 什么是合成层？
- 哪些属性可能触发独立合成层？
- 合成层越多越好吗？

## ⚠️ 易错点 / 高阶拓展
合成层不等于性能一定更好。高阶回答要能提到“减少重绘范围”和“增加内存占用”这两面性。
