# 列表渲染优化

## 📌 核心概念
列表性能瓶颈通常来自节点过多、每项渲染成本高、滚动中频繁更新。优化手段包括虚拟列表、按需分页、稳定 key、拆分行组件、避免每项创建昂贵对象。

## 💻 代码示例
```jsx
function Row({ item }) {
  return <div className="row">{item.name}</div>;
}

const MemoRow = React.memo(Row);

function List({ items }) {
  return items.map((item) => <MemoRow key={item.id} item={item} />);
}
```

## 🛠 实战场景
商品列表、聊天记录、日志面板、运营表格都是高频列表场景，稍不注意就会卡顿。

## 🎯 面试怎么问
- 大列表为什么会卡？
- 虚拟列表的核心思想是什么？
- 除了虚拟化，还有哪些优化手段？

## ⚠️ 易错点 / 高阶拓展
不是所有列表都要上虚拟化。若项目数据量不大、行高复杂、SEO 要求高，分页或服务端渲染可能更合适。
