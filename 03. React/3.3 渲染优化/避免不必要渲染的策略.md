# 避免不必要渲染的策略

## 📌 核心概念
React 性能优化的核心不是“阻止所有渲染”，而是让真正需要更新的组件更新。常见策略包括状态下沉、组件拆分、稳定引用、减少 Context 大范围传播、列表虚拟化。

## 💻 代码示例
```jsx
function Dashboard() {
  return (
    <div>
      <SearchPanel />
      <MemoizedChart />
      <UserTable />
    </div>
  );
}
```

## 🛠 实战场景
大型页面里常见问题是一个搜索框改动导致整棵页面树都跟着重渲染。合理拆分状态边界往往比加一堆缓存更有效。

## 🎯 面试怎么问
- React 中常见的无效渲染来源有哪些？
- 状态应该放在哪一层最合适？
- 为什么说“先拆组件，再谈 memo”？

## ⚠️ 易错点 / 高阶拓展
先做结构性优化，再做局部缓存。盲目加 `useMemo`、`memo` 很容易治标不治本。
