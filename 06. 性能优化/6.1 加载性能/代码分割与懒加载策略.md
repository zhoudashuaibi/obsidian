# 代码分割与懒加载策略

## 📌 核心概念
代码分割是把大包拆成多个 chunk，懒加载则让非首屏代码在真正需要时再加载。两者结合能显著降低首屏下载、解析、执行成本。

## 💻 代码示例
```js
const UserPage = React.lazy(() => import('./UserPage'));

<Suspense fallback={<div>loading</div>}>
  <UserPage />
</Suspense>
```

## 🛠 实战场景
多路由应用、后台系统、组件库文档站都适合做路由级或组件级拆包，避免一次性加载所有业务代码。

## 🎯 面试怎么问
- 代码分割为什么能优化性能？
- 路由懒加载和组件懒加载的差异是什么？
- 分包是否越细越好？

## ⚠️ 易错点 / 高阶拓展
懒加载减少首屏成本，但会增加首次进入该模块时的等待。实际要配合预加载、缓存和骨架屏平衡体验。
