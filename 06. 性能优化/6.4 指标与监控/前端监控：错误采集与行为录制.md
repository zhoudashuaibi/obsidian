# 前端监控：错误采集与行为录制

## 📌 核心概念
前端监控通常覆盖错误、性能、行为三类信息。错误采集帮助定位 JS 异常和资源失败，行为录制帮助还原用户操作上下文，两者结合才能更高效复盘线上问题。

## 💻 代码示例
```js
window.addEventListener('error', (event) => {
  report({
    type: 'js_error',
    message: event.message,
    filename: event.filename,
  });
});
```

## 🛠 实战场景
线上白屏、接口异常、偶现交互 bug 往往本地难复现，这时监控平台和用户行为回放能显著缩短排障时间。

## 🎯 面试怎么问
- 前端监控通常采哪些数据？
- 行为录制为什么有价值？
- 监控上报要注意哪些隐私与性能问题？

## ⚠️ 易错点 / 高阶拓展
监控不是越多越好。采集要控制体积、采样率和敏感数据脱敏，避免监控本身反过来拖慢页面。
