# 重排 Reflow vs 重绘 Repaint 触发与避免

## 📌 核心概念
重排指布局信息变化后浏览器重新计算元素几何位置，成本通常高于只重绘颜色、背景等视觉更新。频繁触发布局会让动画和交互明显卡顿。

## 💻 代码示例
```js
const width = el.offsetWidth; // 读取布局
el.style.width = width + 10 + 'px'; // 写布局
// 读写交错可能触发布局抖动
```

## 🛠 实战场景
拖拽、滚动监听、动画、复杂表格更新时，若频繁读写布局属性，很容易出现掉帧和布局抖动。

## 🎯 面试怎么问
- 重排和重绘有什么区别？
- 哪些操作容易触发重排？
- 如何减少布局抖动？

## ⚠️ 易错点 / 高阶拓展
经典优化思路是批量写入、读写分离、用 transform/opacity 做动画，并减少大范围布局影响。
