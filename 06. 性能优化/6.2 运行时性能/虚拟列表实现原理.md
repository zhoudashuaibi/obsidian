# 虚拟列表实现原理

## 📌 核心概念
虚拟列表只渲染可视区域附近的少量节点，通过容器占位和滚动偏移来模拟完整列表，从而把 DOM 数量控制在较小范围。

## 💻 代码示例
```js
const start = Math.floor(scrollTop / itemHeight);
const end = start + visibleCount;
const visibleItems = list.slice(start, end);
```

## 🛠 实战场景
聊天记录、日志面板、超大表格、商品瀑布流都很适合虚拟化，否则几千上万节点会让渲染和滚动体验急剧下降。

## 🎯 面试怎么问
- 虚拟列表为什么能优化性能？
- 定高和不定高列表实现上有什么区别？
- 虚拟列表有哪些副作用？

## ⚠️ 易错点 / 高阶拓展
虚拟化会增加实现复杂度，并可能影响 SEO、锚点定位、无障碍和测量逻辑，不是所有列表都值得上。
